NeatWhisky · 技术原理

在 Apple Silicon (macOS) 上，绕过 Whisky 停更的 Wine 7.7，解决乱码、闪退、黑屏与关闭重启
Whisky（社区版 · 已停更） Wine Staging 11.x Steam 客户端 CEF 126 Apple M 系列 / macOS 15

本文是 NeatWhisky「为什么这样修」的技术深潜，面向想了解内部机制或排障的人。README 面向小白只讲价值与一键体验，技术细节都在这里。

1. 背景：Whisky 停更带来的连锁问题

Whisky 是 macOS 上的图形化 Wine 前端，已停止维护，内置 Wine 长期停在 Wine 7.7（2022 年）。而 2026 年的 Steam 客户端使用 CEF 126（Chromium 内核），老 Wine 无法承载，于是出现一连串问题：

现象根本原因结果
界面中文乱码（方块/问号）Wine 前缀缺少 CJK 字体与字体链接已解决
启动即闪退 / steamwebhelper 崩溃循环Wine 7.7 无法运行 CEF 126已解决
能启动但界面全黑CEF 126 在 Wine 11 下 GPU 合成绘制为黑窗已解决
关闭后自动重启、关不掉包装器子进程变成孤儿残留已解决

Whisky 只是个外壳，真正干活的是底层 Wine 与独立的 Wine 前缀（bottle）。所以无需换掉 Whisky，只要替换其使用的 Wine、并对前缀做针对性配置，现代 Steam 就能跑起来。

2. 核心思路

老 Wine 7.7 跑不动 CEF 126→ ① 换 Wine 11.x→ 不再闪退→ ② 装 CJK 字体→ 不再乱码→ ③ 强制 CEF 单进程→ 不再黑屏→ ④ 包装器进程归一→ 能正常关闭

四步分别针对四个问题，前一步是后一步的前提。其中第③步是关键：CEF 126 在 Wine 11 下默认的多进程 + GPU 合成会把窗口绘制成全黑；而 Chromium 的 --single-process --disable-gpu 模式可绕开。但 Steam 不会把这两个标志透传给子进程，所以必须用一个包装器（wrapper）替换 steamwebhelper.exe，由它在调用真正内核时强行追加这些标志。

3. 步骤一 · Wine 7.7 → Wine Staging 11.x

1
下载现代 macOS 版 Wine Staging 11.x（如 Gcenx 构建），解压备用。
2
备份并替换 Whisky 共享 Wine 目录（~/Library/Application Support/<bundleid>/Libraries/Wine）。
3
Wine 11 是单一 WoW64 的 wine，而 Whisky 调用 wine64，补符号链接：ln -s wine wine64。
4
修改 WhiskyWineVersion.plist 版本为 11.x，防止被降级回 7.7。
5
用新 Wine 升级既有前缀：wineboot -u。

完成后闪退/崩溃循环消失，但还有乱码与黑屏，继续后续步骤。

4. 步骤二 · 中文乱码

根因是前缀缺 CJK 字体且字体替换/链接没指向有效字体。用 winetricks 安装字体包并自动配置 FontLink：

export WINEPREFIX=<bottle>
winetricks cjkfonts

之后 drive_c/windows/Fonts/ 会新增 sourcehansans.ttc 等，并在 system.reg 中为 Microsoft Sans Serif / Tahoma 建立 FontLink，重启 Steam 后中文正常。

5. 步骤三 · 界面黑屏（steamwebhelper 包装器）

升级 Wine 后能启动但全黑——CEF 126 在 Wine 11 的已知渲染 bug。让 CEF 以 --disable-gpu --single-process 运行即可绕开。由于 Steam 不透传这些标志，编译一个极小的 Windows 程序作为 包装器，替换 steamwebhelper.exe，由它启动真身（改名为 steamwebhelper_orig.exe）并追加标志。

brew install mingw-w64
x86_64-w64-mingw32-gcc swh_wrapper.c -o steamwebhelper_wrapper.exe -mwindows -O2 -municode
for d in cef.win64 cef.win7x64; do
  D="$BOTTLE/drive_c/Program Files (x86)/Steam/bin/cef/$d"
  [ -f "$D/steamwebhelper_orig.exe" ] || mv "$D/steamwebhelper.exe" "$D/steamwebhelper_orig.exe"
  cp steamwebhelper_wrapper.exe "$D/steamwebhelper.exe"
done
验证：启动后查看进程，应只剩约 3 个 steamwebhelper（包装器外壳 + 单进程内核 + crashpad），且内核命令行末尾带 --disable-gpu --single-process。

6. 步骤四 · 关不掉、不停重启

最初的包装器只是简单启动子进程。Steam 想重启界面时只杀外壳，真身子进程变孤儿残留，Steam 误以为界面崩溃便反复拉起——表现为「关不掉、不断重启」。

修复：包装器内使用 Windows Job Object + JOB_OBJECT_LIMIT_KILL_ON_JOB_CLOSE。外壳退出时操作系统连同子进程一起结束，生命周期干净。

7. 步骤五 · 启动参数与防文件还原

-cef-disable-gpu -cef-disable-gpu-compositing -noverifyfiles
参数作用
-cef-disable-gpu / -cef-disable-gpu-compositing关闭 CEF GPU 加速/合成，配合包装器修复渲染
-noverifyfiles关键：阻止 Steam 启动时把包装器还原成原版

8. 维护与自愈

Steam 大更新可能覆盖包装器导致黑屏复发。重新安装包装器即可恢复（见 SteamFix/reapply-wrapper.sh）。NeatWhisky 会在启动 Steam 前自动校验并按需重装。
